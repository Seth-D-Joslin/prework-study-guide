HTML

  • HTML stands for Hypertext Markup Language.
  • HTML is used to give the content STRUCTURE.
  • The [HEAD ELEMENT] gives INFORMATION about the webpage.
  • The [BODY ELEMENT] represents the VISIBLE CONTENT shown to the user.
  • A [SEMANTIC ELEMENT] clearly describes its meaning to BOTH the browser and the developer. ie: header, footer, and summary
  • A [NON-SEMANTIC ELEMENT] is used to structure web layouts and contain content, but they don't provide any information about the content.
  • A [NON-SEMANTIC ELEMENT] is often used to apply JavaScript (JS) actions or Cascading Style Sheets (CSS) styling to content.
  • The [SECTION ELEMENT] defines a section in a document. Note that section is a format that can be used in multiple places within the html.index.
  • The [ALT ATTRIBUTE] AKA [ALT TAG] is used to provide an alternative text description to a link that may or may not be working.
  • Note to self: revise & add more notes about html through index tab in vs code.

CSS

  • CSS stands for Cascading Style Sheets.
  • CSS is used to give the content STYLE.
  • A [MARGIN] indicates how much space we want around the OUTSIDE of an element.
  • A [PADDING] indicates how much space we want around the content INSIDE an element.
  • Note to self: revise & add more notes about css through index tab in vs code.

Git

  • The term 'git' refers to data that is stored to a local computer that can be manipulated by a user.
  • The term 'github' refers to data that is stored to the "cloud" that can be manipulated by all who have access.
  • git status: checks what branch we are currently on
  • git checkout -b branch-name:creates a new branch and switches to it
  • Note to self: revise & add more notes about git through index tab in vs code.

JavaScript

  • JS stands for JavaScript.
  • JS handles LOGIC and allows users to INTERACT with a website.
  • A [VARIABLE] is a named container that allows us to store data in our code.
  • [CONTROL FLOW] is the order in which a computer executes code in a script.
  • Note to self: revise & add more notes about JavaScript through index tab in vs code.
